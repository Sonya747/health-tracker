## 4. APK 构建

### 构建方式（本机已配好环境）

```bash
cd /Users/admin/projects/Tally/apps/mobile
npx expo prebuild --platform android --no-install   # 已执行过，android/ 已存在
cd android
export JAVA_HOME=/opt/homebrew/opt/openjdk@17
export ANDROID_HOME=$HOME/Library/Android/sdk

```

产物路径：`apps/mobile/android/app/build/outputs/apk/release/app-release.apk`

## 4.5 代码复查与修复

构建期间由独立复查发现并已修复 5 个问题（commit `bb50bd7`）：

1. 日历弹窗常驻不卸载，重新打开时停留在旧月份 → 打开时重置到当前选中日期所在月。
2. 个人状态「只记标签未打分」的天数在周/月统计中被算作无记录 → `recordedDays` 按有记录天数统计，均分只按打分的天算（补 2 个单测）。
3. App 过夜后回前台，`selectedDate` 停留在昨天，计数会写错日期 → AppState 监听跨天自动跟进（用户主动选择的历史日期不受影响）。
4. 计数手动编辑时空输入会被静默存成 0 → 空输入报校验错误，清零需点「清零」按钮。
5. 通过 deep link 打开记录表单时，URL 的 date 参数与实际写入日期可能不一致 → 表单挂载时对齐 store 日期。

## 5. 手动验收步骤

安装：将 APK 传到 Android 手机（系统分享/数据线/adb `adb install app-release.apk`），允许安装未知来源应用。

1. **首次启动**：打开 App，应直接进入「今日」页，顶部显示今天日期（本地时区），底部有 今日 / 统计 / 设置 三个 Tab。
2. **计数**：点排便「＋」两次 → 数字变 2；点「−」一次 → 变 1；数字为 0 时「−」置灰。点数字或「编辑」→ 手动输入 5 保存 → 显示 5；输入 -1 或 100 → 显示校验错误。
3. **睡眠**：点「记录睡眠」→ 选开始 23:00、结束 07:00 → 时长自动显示 8小时 → 选质量「较好」→ 清醒次数 +2 → 清醒时长选 30分钟 → 深睡比例填 25 → 勾选标签「早醒」「再次入睡困难」并自定义添加一个标签 → 填备注 → 保存 → 今日页睡眠卡片显示时长、清醒 2 次、深睡 25% 和标签。再点「编辑」改为只填时长 7小时 → 保存生效。只填开始不填结束 → 保存时提示错误；清醒时长选成超过睡眠总时长 → 提示错误；深睡比例填 101 → 提示错误。
4. **个人状态**：点「记录状态」→ 什么都不填直接保存 → 提示至少填一项；选两个标签 + 评分 4 → 保存 → 今日页显示标签和评分。
5. **焦虑发作**：新增一条（时间 10:30、时长 15分钟、强度中等、诱因「工作压力」）→ 保存；再新增第二条 → 今日页显示「焦虑发作（2 次）」列表；点第一条 → 修改强度 → 保存生效；点「删除」→ 出现确认弹窗 → 确认后消失。
6. **今日备注**：输入文字 → 点「保存备注」→ 显示「已保存」。
7. **日期切换**：点「‹」到昨天 → 各卡片显示昨天（应为空状态，文案清晰）→ 补录一条排便 → 点日期打开日历 → 点「回到今天」→ 今天数据仍在。
8. **持久化（关键）**：完全杀掉 App（最近任务里划掉）→ 重新打开 → 上面所有记录仍在。
9. **统计-日**：统计页默认「日」，显示今天所有记录明细。
10. **统计-周**：切「周」→ 排便/排尿/睡眠柱状图出现，今天对应柱子有值；点某天柱子 → 进入当日详情。
11. **统计-月**：切「月」→ 汇总数字正确，「记录覆盖」里有记录的日期高亮；点昨天 → 显示昨天补录的排便记录。
12. **空状态**：切到没有记录的上一周/上一月 → 显示「这个时间段还没有记录」。
13. **导出**：设置 → 导出 JSON → 系统分享面板弹出 → 保存到「文件」→ 用文本查看，包含 categories/records/dailyNotes；导出 CSV → Excel/WPS 打开，中文正常、逗号引号备注不串列。
14. **隐私说明**：设置 → 隐私说明 → 五段说明完整，明确「数据仅保存在本机」。
15. **离线**：开飞行模式，重复 2-13 步 → 一切正常（App 不依赖网络）。

## 6. 已知限制 / 后续建议

- release APK 为 debug keystore 签名，仅供个人安装。
- 时间/时长/日历选择器为自研轻量组件（避免额外原生依赖），非系统原生滚轮样式。
- 图表为轻量自绘柱状图（开发计划允许「列表 + 简单柱状图」），月视图 31 根柱子在窄屏上较密，可点击柱子看详情。
- 表单校验用 `packages/core` 的纯函数实现（未引入 React Hook Form + Zod，减少依赖；校验规则位置与计划一致，在 core 层，跨端可复用）。
- APK 体积 98MB：Expo/RN 通用 release 包含四种 ABI 的原生库；后续可按 ABI 拆分或开启 minify 减小体积。
- 后续路线：自定义类别 → 自定义表单项 → App 锁/加密 → Taro 小程序端（见 `docs/miniapp-integration.md`）。
